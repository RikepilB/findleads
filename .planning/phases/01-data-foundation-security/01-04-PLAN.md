## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Future scraped Places content -> `upsertBusiness`/`insertLeadSnapshot` parameters | These two functions are the exact seam where untrusted external data (Phase 2+) will enter the database |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Tampering | A generic "update every column from `excluded.*`" upsert helper accidentally clobbering `notes`/`contacted`/`firstSeenAt` on `businesses` | high | mitigate | `upsertBusiness`'s `set` object is hand-written and explicitly omits the three CRM-preserving columns; Task 2's automated verify greps for their accidental presence inside the `onConflictDoUpdate` block |
| T-04-02 | Repudiation | Worker retry/resume producing duplicate or erroring `leads` rows for the same `(jobId, placeId)` pair | medium | mitigate | `insertLeadSnapshot` uses `onConflictDoNothing` (a declarative, race-free Postgres primitive) rather than a try/catch around a plain insert, which this repo's own guardrails forbid as silent error swallowing |
</threat_model>
