| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-------------------|
| T-02-03-01 | Tampering | `searchTextPlaces`'s response handling | medium | mitigate | Raw JSON response validated via `textSearchResponseSchema.parse(...)` before any field is read; malformed/renamed/wrong-typed fields throw immediately rather than propagating an `any`-typed surprise into `mapPlaceToLead.ts` (Plan 02-04) |
| T-02-03-02 | Information Disclosure | `PlacesApiError` / error paths | high | mitigate | `PlacesApiError` carries only `status` and response body text — the outbound request object (which contains the `X-Goog-Api-Key` header) is never included in any thrown error's message or properties; regression-tested directly in this plan's Task 1 |

| T-02-03-04 | Tampering (SSRF-adjacent) | `TEXT_SEARCH_URL` constant | low | accept | Hardcoded literal string, never templated from `category`/`location`/`pageToken` or any other external input — there is no way user-controlled input can redirect the request to a different host |
