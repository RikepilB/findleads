    - `fetchPage` stub that throws `new Error('INVALID_REQUEST: token not active')` on its first two

    - `fetchPage` stub that always throws the token-not-yet-active error, exhausting
      `DEFAULT_PAGE_TOKEN_RETRY.maxAttempts` (3): `fetchNextPage` rethrows the last error — it does
      not swallow it or return `undefined`.
    - `fetchPage` stub that throws a DIFFERENT error (e.g. `new Error('RATE_LIMITED')`) on its first
      call: `fetchNextPage` rethrows that error immediately, without retrying (it is not a
      token-not-yet-active condition, per `isTokenNotYetActiveError`'s `message.includes
      ('INVALID_REQUEST')` check).
    - A custom `RetryConfig` (e.g. `{ maxAttempts: 1, delayMs: 10 }`) passed explicitly overrides

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Free-text location string → `locale.ts`'s regex matching | Arbitrary-shaped input matched against a small, fixed, developer-controlled rule table; output is always one of a small fixed set of locale pairs, never derived from or echoing the input itself |
| `paginate.ts`'s injected `fetchPage` callback → its thrown errors | `fetchNextPage` distinguishes "token not yet active" from all other errors solely via one string match on the error message; a caller's error shape is trusted only for that one narrow check |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-------------------|
| T-02-02-01 | Tampering | `lib/places/locale.ts` (`LOCALE_RULES` regex matching) | low | accept | Pure lookup against a fixed, developer-controlled rule list; attacker-controlled location text can only select among a small set of developer-defined locale pairs or fall to the default — no injection surface exists since the output is always one of those literal pairs, never derived from the input string itself |
| T-02-02-02 | Denial of Service | `lib/places/paginate.ts` (`fetchNextPage` retry loop) | low | mitigate | Bounded by an explicit `maxAttempts` (default 3); non-token-related errors are rethrown immediately rather than retried — no unbounded retry storm is possible regardless of caller input |
| T-02-02-SC | Tampering | npm/pip/cargo installs | n/a | n/a | No new packages are installed in this plan |
</threat_model>
